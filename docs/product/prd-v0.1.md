# 画布主导待办产品 PRD v0.1

更新时间：2026-03-12

## 1. 产品定位

一句话定位：

`一个以左侧主画布承载上下文、以右侧待办列表承载行动投影的 AI 视觉待办产品。`

用户只需要复制内容并按一下快捷键，信息就进入左侧画布；系统再从这些原始信息中提取待办，并投影到右侧列表。

应用默认应支持后台常驻。收录动作发生时，主窗口不要求已经打开，系统只需在电脑右下角给出一个轻量提示，例如“正在整理并记录到画布”。

这不是传统待办工具，也不是白板换皮产品。它的核心价值不是“把任务录进系统”，而是“在不丢失来源上下文的前提下，把行动项自然地长出来”。

## 2. 问题定义

当前主流待办工具默认要求用户：

- 自己打字录入。
- 在录入当下做分类、命名、设时间、设优先级。
- 维护一个长期稳定的结构化系统。

但目标用户的真实痛点是：

- 信息来源太多，复制、转发、录入都很费心。
- 录入时要做太多判断：记不记、怎么记、要不要设时间。
- 很多输入内容是“半待办、半参考、半提醒”，不适合强结构化。
- 一旦只留下一个待办标题，消息原文、截图、语气和上下文很快就丢了。

## 3. 目标用户

核心用户画像：

- 信息输入密度高的人。
- 偏视觉思考、非线性整理的人。
- 不喜欢高纪律 GTD，但又怕漏事的人。
- 习惯先收纳、后整理的人。
- 接受截图、便签、碎片信息混杂工作方式的人。

典型角色：

- 产品经理
- 创作者 / 内容工作者
- 自由职业者
- 小团队负责人
- 容易被信息流打断的人

## 4. 用户任务（JTBD）

核心 Job：

`当我看到一条可能需要后续处理的信息时，我希望几乎不思考地把它收进画布，并让系统自动把行动项投影到待办列表里，这样我既不会丢掉上下文，也不用当场完成复杂录入。`

子任务：

- 快速捕获文本、截图、图片。
- 保留原始上下文。
- 自动抽取可能的待办、时间线索、标签。
- 在右侧快速浏览可执行事项。
- 在左侧随时回到来源和背景信息。

## 5. 产品原则

1. Capture First
   先收，再理。任何增加收录阻力的设计都要谨慎。

2. Background by Default
   应用应默认可以在后台接收收录动作，而不是要求用户先打开主界面。

3. Source First
   左侧画布是来源真相层，原文和截图必须保留。

4. Action Is Projection
   右侧待办不是独立录入系统，而是来源内容的行动投影。

5. AI Suggests, User Decides
   AI 只做建议，不做不可逆强控制。

6. Link Must Be Visible
   用户看待办时，必须能快速知道“这件事是哪来的”。

## 6. 核心使用场景

### 场景 A：聊天消息转成待办

用户复制一句“下周三之前给我一个初稿”并按快捷键。

系统行为：

- 应用即使未打开主窗口，也能在后台接收本次收录。
- 电脑右下角出现轻量提示，告诉用户“已记录，正在整理”。
- 左侧主画布生成来源卡片。
- AI 提取“可能待办”“可能时间：下周三前”“可能对象：初稿”。
- 右侧生成一个待确认待办条目。
- 用户鼠标移到右侧条目时，左侧对应来源卡片高亮并出现连线。

### 场景 B：截图留存

用户截了一张聊天图或活动海报，不想手打说明。

系统行为：

- 图片进入左侧画布。
- AI 生成简短摘要和可能行动项。
- 如果存在行动项，右侧生成相应待办。
- 用户从右侧查看待办时，能立刻看到这件事对应的图片证据。

### 场景 C：回看待办时找上下文

用户在右侧看到“跟进海报修改”，但已经忘了是哪里来的。

系统行为：

- hover 该待办。
- 左侧自动高亮对应的聊天截图或文本来源。
- 点击待办，左侧画布自动定位到相关来源区域。

## 7. 差异化

### 我们不是

- 不是普通列表待办。
- 不是单纯的无限白板。
- 不是把图片附件挂在任务上的老式任务工具。

### 我们是

- 一个左画布右列表的双栏工作区。
- 一个“上下文和行动同时存在”的待办产品。
- 一个让待办条目永远能回指来源证据的系统。

### 核心差异点

- 输入动作极短：复制后一个快捷键。
- 原始上下文保留在左侧画布，不需要二次转述。
- 待办是从来源中投影出来的，不是重新录入的。
- 右侧负责执行效率，左侧负责上下文完整性。
- hover / click 联动是关键体验，而不是附属动画。

## 8. MVP 范围

### 必做

- 桌面优先应用壳
- 后台常驻运行
- 系统托盘入口
- 全局快捷键
- 读取剪贴板文字
- 读取剪贴板图片 / 截图
- 右下角轻量通知
- 左侧主画布
- 右侧待办列表
- 自动生成来源卡片并落位
- AI 异步提取：
  - 简短摘要
  - 待办建议
  - 时间线索
  - 轻量标签
- 从来源生成待办投影
- 待办与来源的双向关联
- hover 待办高亮左侧来源并显示连线
- 点击待办自动定位左侧来源
- 搜索
- 本地优先存储

### 延后

- 多人协作
- 云同步
- 浏览器扩展
- 重度提醒与重复任务
- 复杂项目管理
- 移动端优先

## 9. 成功指标

- 从复制到成功收录，P50 小于 2 秒。
- 用户不打开主窗口也能完成一次收录。
- 至少 70% 的新输入不需要用户重打标题。
- 用户能在右侧待办 hover 时快速理解来源。
- 用户在一周内至少发生一次“从右侧待办回到左侧来源”的行为。

## 10. 核心风险

1. 如果左右两栏没有真实联动，产品会退化成“白板 + 普通待办侧栏”。
2. 如果 AI 提取不准，右侧待办会失去可信度。
3. 如果左侧画布没有默认弱结构，用户会觉得杂乱。
4. 如果右侧列表过强，左侧画布会沦为附件区。

## 11. 当前产品判断

当前这个产品的第一性原理是：

`低摩擦捕获 + 以画布保留上下文 + 以列表投影行动 + AI 轻整理`
